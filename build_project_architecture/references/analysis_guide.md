# 源码架构分析指南

## 分析方法论

### 1. 自顶向下分析法

从模块的入口类开始，逐层深入理解：

1. **识别入口点** — 找到 Main 类、Catalog 类、Application 类等顶层入口
2. **理解接口契约** — 阅读接口定义，理解模块对外提供的能力
3. **追踪实现** — 从接口到实现类，理解核心逻辑
4. **梳理依赖** — 找出外部依赖和内部模块间的依赖关系

### 2. 场景驱动分析法

从用户使用场景出发，追踪代码执行路径：

1. **列举核心场景** — 创建、读取、更新、删除等基本操作
2. **断点式追踪** — 从入口方法开始，逐步追踪方法调用
3. **记录调用链** — 记录每一步的类名、方法名、参数和返回值
4. **标注关键逻辑** — 标注条件判断、异常处理、状态转换等关键点

## 设计模式识别速查

### 创建型模式

| 模式 | 识别特征 | 常见表现 |
|------|---------|---------|
| Factory Method | 返回接口/抽象类型的 create/build 方法 | `createXxx()` 返回抽象类型 |
| Abstract Factory | 一组工厂方法返回相关对象族 | 多个 `createXxx()` 方法在同一接口中 |
| Builder | 链式调用 + `build()` 终结方法 | `.withXxx().withYyy().build()` |
| Singleton | 私有构造器 + 静态获取方法 | `getInstance()`, `INSTANCE` 字段 |

### 结构型模式

| 模式 | 识别特征 | 常见表现 |
|------|---------|---------|
| Adapter | 将一个接口转换为另一个接口 | 包装类实现目标接口，内部委托原始对象 |
| Decorator | 包装同类型对象，增强功能 | 实现相同接口，构造器接受同接口参数 |
| Facade | 简化复杂子系统的入口 | 高层类组合多个底层组件 |
| Proxy | 控制对目标对象的访问 | 实现相同接口，延迟加载或添加控制逻辑 |
| Composite | 树形结构，叶节点和组合节点统一接口 | 接口含 `add/remove/getChildren` |

### 行为型模式

| 模式 | 识别特征 | 常见表现 |
|------|---------|---------|
| Strategy | 运行时选择算法 | 接口定义算法，多种实现类 |
| Template Method | 抽象类定义骨架，子类实现步骤 | 抽象类中的 `final` 方法调用 `abstract` 方法 |
| Observer | 事件通知机制 | `addListener/register` + 回调方法 |
| Iterator | 遍历集合元素 | `hasNext()/next()` 或 `Iterable` 实现 |
| Chain of Responsibility | 请求沿链传递 | `handler.setNext()` 或 `List<Handler>` |
| Command | 封装请求为对象 | `execute()` 方法的请求对象 |
| State | 状态驱动行为变化 | 枚举状态 + 状态转换逻辑 |
| Visitor | 双分派操作 | `accept(Visitor)` + `visit(Element)` |

### 并发模式

| 模式 | 识别特征 | 常见表现 |
|------|---------|---------|
| Optimistic Locking | 版本号/ETag 检查 | 先读后写，写时检查版本是否变化 |
| Retry | 失败重试 | `while/for` 循环包裹操作 + 异常捕获 |
| Immutable Object | 所有字段 `final`，无 setter | 修改产生新实例 |

## 架构图绘制指南

### 分层架构图

典型的分层从上到下：

```
┌─────────────────────────────────┐
│         用户接口层 / API 层       │  ← 公共接口、网关
├─────────────────────────────────┤
│         业务逻辑层 / 核心层       │  ← 核心业务处理
├─────────────────────────────────┤
│         数据访问层 / 存储层       │  ← 数据持久化
├─────────────────────────────────┤
│         基础设施层              │  ← 工具类、配置、通信
└─────────────────────────────────┘
```

### 流程图标准符号

- **椭圆**: 开始/结束
- **矩形**: 处理步骤
- **菱形**: 判断/决策
- **平行四边形**: 输入/输出
- **箭头**: 流程方向

### 时序图要素

- **参与者**: 顶部矩形框
- **生命线**: 参与者下方的虚线
- **同步消息**: 实线箭头 →
- **返回消息**: 虚线箭头 ⇢
- **自调用**: 弯曲箭头回到自身
