## 注释规则速查

### 必须跳过

- `package` 行
- `import` 行
- 单元测试代码
- 不超过 5 行的方法体内部逐行注释
- 变量声明行
- 变量引用行
- 极简单算数运算逻辑
- 已有中文注释的代码行或函数

### 应优先注释

- 类、接口、枚举的职责
- 方法的作用
- 关键条件分支
- 循环目的
- 异常处理与兜底逻辑
- 状态切换
- 资源申请与释放
- 序列化、反序列化、校验、转换
- 复杂链式调用的整体目的

## 决策表

### 情况 1：`package` / `import`

处理：直接跳过，不加注释。

### 情况 2：测试代码

处理：直接跳过，不加注释。

判断信号示例：

- 文件路径位于 `src/test`
- 类名包含 `Test`
- 方法使用测试框架注解

### 情况 3：方法体不超过 5 行

处理：只写方法作用，不写方法体内部逐行注释。

示例：

```java
// 返回当前算子的名称
public String getOperatorName() {
    return operatorName;
}
```

### 情况 4：变量声明

处理：默认跳过。

示例：

```java
int size = list.size();
```

除非该变量声明承载关键业务动作，否则不需要写：

```java
// 定义一个整型变量 size
int size = list.size();
```

### 情况 5：变量引用

处理：默认跳过。

示例：

```java
return value;
process(value);
```

### 情况 6：简单算数逻辑

处理：默认跳过。

示例：

```java
index++;
count = count + 1;
```

### 情况 7：复杂条件分支

处理：在分支前补充意图说明。

示例：

```java
// 如果当前任务已经取消，则直接终止后续处理
if (isCanceled) {
    return;
}
```

### 情况 8：多步逻辑块

处理：优先对整个逻辑块写概括注释，而不是逐行拆散。

示例：

```java
// 先校验配置，再创建运行所需的上下文对象
validate(config);
Context context = createContext(config);
start(context);
```

### 情况 9：复杂链式调用

处理：解释最终目标，不解释每一级调用。

示例：

```java
// 从执行环境中提取配置并转换为内部可复用的运行参数
RuntimeConfig runtimeConfig =
        executionEnvironment.getConfiguration().toMap().entrySet().stream()
                .collect(Collectors.toMap(Map.Entry::getKey, Map.Entry::getValue));
```

## 推荐注释风格

- 使用简体中文。
- 优先使用单行注释放在代码上方。
- 避免写成"定义变量""调用方法""进入 if 判断"这类废话。
- 优先解释"为什么这样做"与"这段逻辑的职责"。
- 保持术语前后一致，例如"运行时上下文""检查点状态""序列化器"等不要来回换叫法。

## 自检清单

提交修改前执行以下自检：

- 是否误给 `import` 或 `package` 加了注释？
- 是否误给测试代码加了注释？
- 是否误给短函数的函数体逐行加了注释？
- 是否误给变量声明、变量引用、简单算数表达式加了注释？
- 是否误给已有中文注释的代码行或函数重复添加注释？
- 注释是否真的帮助理解逻辑，而不是复述代码表面含义？
- 注释是否简洁，没有明显噪声？

## 文件处理标记

为确保每个文件只被处理一次，技能采用以下标记机制：

1. **完成标记**：在处理完成的文件开头添加一行注释：`// 这个文件已经全部加上中文注释`（对于Scala/Java文件使用`//`，其他语言使用对应的单行注释语法）。
2. **跳过检查**：处理文件前检查文件开头是否已有完成标记，如果有则跳过该文件，避免重复处理。
3. **标记位置**：标记应添加在文件最前面，位于任何代码之前（包括package声明之前）。
